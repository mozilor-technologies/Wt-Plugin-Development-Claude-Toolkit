# context-save-plan

Save the current session plan to `.context/plans/` and activate it for git commit tracking.

## What This Skill Does

1. Asks the user for a short name for the plan
2. Creates `.context/plans/YYYY-MM-DD-<name>.md` with the plan content
3. Writes the filename to `.context/plans/.current` (making it the active plan)
4. Regenerates `.context/plans/index.md`
5. Makes an initial git commit of the new plan file

## Execution Steps

### Step 1: Verify Setup

Run `git rev-parse --show-toplevel` to find `REPO_ROOT`.

Check that `$REPO_ROOT/.context/plans/` exists. If not, tell the user:
> "`.context/plans/` not found. Please run `/context-init` first to set up the context folder."

### Step 2: Collect Plan Name

Ask:
> "What is a short name for this plan? (e.g. `add-auth`, `fix-login-bug`, `refactor-api`)"

Sanitize the input: lowercase, replace spaces with hyphens, strip special characters. Validate it is non-empty and kebab-case.

### Step 3: Determine Plan Content

Check if the current conversation contains a written plan (look for headings like `# Plan:`, `## Objective`, `## Steps`). If found, confirm with the user:
> "I'll save the plan we discussed. Is that correct? [Y/n]"

If no plan is visible in context, ask the user to paste or describe the plan content.

### Step 4: Collect Missing Fields

If developer name and objective are not already in the plan content, ask:
- "Your name (for the plan header):"
- "One-sentence objective for this plan:"

### Step 5: Write the Plan File

Build the filename: `YYYY-MM-DD-<name>.md` using today's date.
File path: `$REPO_ROOT/.context/plans/YYYY-MM-DD-<name>.md`

Write using the canonical format:

```markdown
# Plan: <Human-readable title derived from name>
**Date:** YYYY-MM-DD
**Status:** in-progress
**Developer:** <name>

## Objective
<one-sentence objective>

## Steps
<plan steps formatted as checkboxes>

## Session Notes

## Commits
```

### Step 6: Write .current Pointer

Write just the filename (e.g. `2026-03-24-add-auth.md`) to:
`$REPO_ROOT/.context/plans/.current`

### Step 7: Regenerate index.md

Scan all `*.md` files in `.context/plans/` (excluding `index.md`). For each, extract `**Status:**` and `**Date:**` from the file header. Write `index.md`:

```markdown
# Plans Index

_Auto-generated by context-save-plan. Do not edit manually._

| Plan | Status | Date |
|------|--------|------|
| [2026-03-24-add-auth.md](2026-03-24-add-auth.md) | in-progress | 2026-03-24 |
```

Sort rows newest first.

### Step 8: Initial Commit

```bash
git add "$REPO_ROOT/.context/plans/YYYY-MM-DD-<name>.md" \
        "$REPO_ROOT/.context/plans/.current" \
        "$REPO_ROOT/.context/plans/index.md"
git commit -m "plan: start YYYY-MM-DD-<name> [plan:YYYY-MM-DD-<name>]"
```

### Step 9: Confirm

Tell the user:

```
Plan saved and activated:
  File:   .context/plans/YYYY-MM-DD-<name>.md
  Active: .context/plans/.current → YYYY-MM-DD-<name>.md

From now on:
  - Each git commit will append a session note and record its hash in this plan
  - Commit messages will automatically include [plan:YYYY-MM-DD-<name>]
  - Use /context-load-plan <name> to reload this plan in a future session
  - Use /context-find-plan <hash> to find this plan from any commit hash
```
